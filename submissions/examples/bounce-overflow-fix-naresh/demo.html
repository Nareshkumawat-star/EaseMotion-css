A11y & Layout Bug Fix

Bounce Animation Clipping

Documenting and resolving the parent overflow: hidden boundary clip issue for 3D translation animations.

Clipped Bounce (The Bug)

The bouncing element hits the container's overflow: hidden upper boundary and is clipped, momentarily disappearing.

Clip ❌
/* Parent is strictly clipping */
.parent {
  overflow: hidden;
}

/* Child animates outside */
.child {
  animation: ease-kf-bounce 1s;
}

Fix 1: Padding Clearance

Adding top padding to the container gives the bouncing element enough internal height to complete its cycle without clipping.

Clear ✨
/* Provide top clearance */
.parent {
  overflow: hidden;
  padding-top: 35px; /* Bounce height */
}

.child {
  animation: ease-kf-bounce 1s;
}

Fix 2: Boundary Offset

Using a negative margin paired with padding expands the parent's overflow region upwards without changing the layout position.

Offset 🚀
/* Expand clipping box upwards */
.parent {
  overflow: hidden;
  margin-top: -35px;
  padding-top: 35px;
}

.child {
  animation: ease-kf-bounce 1s;
}

Technical Deep-Dive

Why does this happen?

When a parent element has overflow: hidden (or overflow: auto / scroll), it establishes a new block formatting context (BFC) and defines a strict boundary for its visual contents.

Even though EaseMotion CSS animations use high-performance, GPU-composited 3D transforms (like translate3d or translateY), the browser must still conform to the CSS spec and clip any rendered content that exceeds the parent boundary container box.

Spec Clarification

Hardware-accelerated rendering on the compositor layer optimizes performance and prevents layout recalculations (jank), but it does not bypass overflow constraints or create an infinite drawing plane outside parent scroll/hidden frames.

Best Practices & Guidelines

  • Check parent styling: When applying vertical motion utilities (like .ease-bounce, .ease-slide-up, or .ease-float), ensure parent elements don't restrict overflow unless necessary.
  • Use the padding solution: For structured layouts (e.g. grids or lists), adding top padding creates the safest design buffer.
  • Use negative margin offsets: For tight, fixed-height header layouts, the negative margin + positive padding technique expands the clipping plane upwards without shifting relative siblings.
  • A11y Considerations: For users with vestibular motion sensitivities, ensure keyframes respect prefers-reduced-motion queries (handled globally in EaseMotion CSS).
